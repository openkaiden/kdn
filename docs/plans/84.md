# Implementation Plan: Podman Runtime Configuration (Issue #84)

## Context

Issue #84 requires making the Podman runtime configurable instead of using hardcoded values. Currently, `pkg/runtime/podman/create.go` hardcodes the base image (Fedora latest), packages, sudoers configuration, and Claude installation. This plan introduces a configuration system that:

- Stores configuration in `~/.kortex-cli/runtimes/podman/config/` (or similar when KORTEX_CLI_STORAGE is defined)
- Separates base image config (`image.json`) from agent-specific config (`claude.json`, `goose.json`, etc)
- Auto-generates default configurations when missing
- Follows existing codebase patterns (interface-based design)

**Configurable:**
- Fedora version (e.g., "latest", "40", "41")
- Packages to install via DNF
- Sudo-authorized binaries
- Custom RUN commands
- Agent installation details

**Hardcoded (not configurable):**
- Base image registry: `registry.fedoraproject.org/fedora`
- User/group
- Sudoers file structure (only binary list is configurable)

## Configuration Schema

### image.json (Base Image Configuration)

Base image registry is hardcoded as `registry.fedoraproject.org/fedora`, only the version tag is configurable.

```json
{
  "version": "latest",
  "packages": ["which", "procps-ng", "wget2", "@development-tools", "jq", "gh", "golang", "golangci-lint", "python3", "python3-pip"],
  "sudo": ["/usr/bin/dnf", "/bin/nice", "/bin/kill", "/usr/bin/kill", "/usr/bin/killall"],
  "run_commands": []
}
```

### claude.json (Agent-Specific Configuration)
```json
{
  "packages": [],
  "run_commands": [
    "curl -fsSL --proto-redir -all,https --tlsv1.3 https://claude.ai/install.sh | bash",
    "mkdir /home/claude/.config"
  ],
  "terminal_command": ["claude"]
}
```

## Implementation Steps

### 1. Create Config Package (`pkg/runtime/podman/config/`)

**config.go**:
- `Config` interface with methods:
  - `LoadImage() (*ImageConfig, error)`
  - `LoadAgent(agentName string) (*AgentConfig, error)`
  - `GenerateDefaults() error`
- Unexported `config` struct implementation
- Factory: `NewConfig(configDir string) (Config, error)`
- Error types: `ErrInvalidPath`, `ErrConfigNotFound`, `ErrInvalidConfig`
- Compile-time check: `var _ Config = (*config)(nil)`

**types.go**:
- `ImageConfig` struct with fields: Version (string), Packages ([]string), Sudo ([]string), RunCommands ([]string)
- `AgentConfig` struct with fields: Packages ([]string), RunCommands ([]string), TerminalCommand ([]string)
- All with JSON tags

**defaults.go**:
- `defaultImageConfig() *ImageConfig` - Current hardcoded values
- `defaultClaudeConfig() *AgentConfig` - Current Claude installation
- Constants for default values

**validation.go**:
- `validateImageConfig(cfg *ImageConfig) error`
- `validateAgentConfig(cfg *AgentConfig) error`
- Validation rules:
  - Version cannot be empty
  - Packages can be empty (valid use case)
  - Sudo binaries must be absolute paths
  - Terminal command must have at least one element

### 2. Integrate with Podman Runtime

**pkg/runtime/podman/podman.go**:
- Add `config config.Config` field to `podmanRuntime` struct
- Modify `Initialize()` method:
  1. Create config directory: `filepath.Join(storageDir, "config")`
  2. Create config instance: `config.NewConfig(configDir)`
  3. Generate defaults: `config.GenerateDefaults()`
  4. Store in struct field

**pkg/runtime/podman/create.go**:
- Modify `createContainerfile()` to accept `*ImageConfig` and `*AgentConfig` parameters
- Replace hardcoded values with config values:
  - Base image: `registry.fedoraproject.org/fedora:{imageConfig.Version}`
  - Packages: merge `imageConfig.Packages` + `agentConfig.Packages`
  - User/group: hardcoded as `claude:claude` (not configurable)
  - Sudo binaries: from `imageConfig.Sudo`
  - RUN commands: from `imageConfig.RunCommands` + `agentConfig.RunCommands`
- Modify `Create()` to load configs before calling `createContainerfile()`
  - **Agent parameter**: For now, hardcode the agent name as `"claude"` when calling `config.LoadAgent()`. In a future update, the agent name will be passed from the `init` command to the runtime's `Create()` method.

**pkg/runtime/podman/containerfile.go** (new):
- Extract Containerfile generation logic
- `generateContainerfile(imageConfig, agentConfig) (string, error)`
- `generateSudoers(sudoBinaries []string) string` - Generate sudoers file from list of allowed binaries
- Build content dynamically from configs
- Hardcode user/group as `claude:claude`
- Generate sudoers aliases and rules from sudo binaries list
